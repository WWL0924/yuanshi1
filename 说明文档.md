# 项目说明文档

## 1. 使用了哪些 AI 工具

本项目开发过程中使用了 AI 编码助手进行方案整理、页面结构搭建、样式实现、权限逻辑补充和文档整理。

本次实际使用的工具是：

- Codex（GPT-5 coding agent）

## 2. 哪些代码是 AI 辅助生成的？做了哪些修改

AI 辅助完成了以下内容：

- React + Vite 项目骨架初始化
- 登录页与主页面结构搭建
- 工单表格组件实现
- ECharts 柱状图实现
- 管理员 / 普通用户权限逻辑
- 页面样式与响应式布局
- README 与交付文档整理

在 AI 生成基础上，进行了以下明确调整与确认：

- 根据需求文档，将用户名 `admin` 设为管理员身份，其他用户名设为普通用户
- 确保普通用户不可见删除按钮
- 确保删除工单后图表自动同步更新
- 增加权限提示区，让管理员与普通用户的能力差异在页面中直观看到


## 3. 项目中最满意或最难处理的点

最满意的点是页面把“表格 + 图表联动 + 权限控制”这三个要求整合成了一个清晰的主流程：

- 登录决定角色
- 角色决定是否可删除
- 删除结果立即反馈到图表

最需要处理清楚的点是权限表达方式。  
不仅要在代码逻辑上限制普通用户删除工单，还要在页面展示上让审核者一眼看出两种角色的区别，所以额外补充了角色标签和权限提示信息。
